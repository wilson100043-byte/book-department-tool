# Wiley 已購買與系所比對工具

這是純前端 GitHub Pages 小工具。使用者不需要安裝 Python、Docker、App，也不需要 API Key。

## 使用方式

1. 開啟 `index.html` 或 GitHub Pages 網址。
2. 上傳 Wiley 全庫 Excel。
3. 選擇待比對工作表，預設 `All Live`。
4. 選擇已購買工作表，預設 `TAEBDC複本202512`。
5. 如果已購買資料是另一個 Excel，可在「已購買 Excel（選填）」另外上傳。
6. 輸入學校名稱。
7. 上傳或貼上系所關鍵字 CSV，可先用 `系所關鍵字範本.csv` 測試。
8. 按「開始比對並產生 Excel」。
9. 下載結果 Excel。

## 輸出

輸出一個工作表：`全部比對結果`。

原始欄位全部保留，只新增：

- `是否已購買`
- `系所`

輸出 Excel 會設定篩選箭頭，可在 Excel 中篩選 `是否已購買 = 未購買`。

## 系所關鍵字 CSV 格式

```csv
學校名稱,系所名稱,關鍵字
淡江大學,財務金融學系,財務金融;金融;投資;corporate finance;investment
淡江大學,會計學系,會計;審計;稅務;accounting;auditing
淡江大學,電機工程學系,電機;電子;半導體;electrical engineering;electronics
```

可直接複製或修改資料夾中的 `系所關鍵字範本.csv`。

## AI 產生系所與關鍵字 Prompt

工具本身不串 API。建議先把系所資料丟到 ChatGPT、Gemini 或其他 AI 聊天工具，請 AI 回傳固定 CSV，再貼回工具。

### 系所清單整理 Prompt

```text
你是一位台灣大學系所資料整理助理。請把我提供的學校、學院、系所或自然語言描述，整理成可貼到書單比對工具的 CSV。

輸出規則：
1. 只輸出 CSV，不要 Markdown，不要說明文字。
2. 第一列固定為：學校名稱,系所名稱,關鍵字
3. 每一列只放一個系所。
4. 學校名稱必須完全使用我指定的學校名稱。
5. 系所名稱請使用完整正式名稱；如果我只給簡稱，請合理補成正式系所名稱。
6. 關鍵字用半形分號 ; 分隔。
7. 每個系所至少產生 8 個關鍵字，包含：
   - 系所全名
   - 常見簡稱
   - 中文主題詞
   - 相關英文學科詞
   - Wiley 書目標題中可能出現的英文詞
8. 不要加入我沒有指定學校的系所。
9. 不要合併不同系所。

我要整理的學校名稱：
【在這裡填學校名稱，例如：淡江大學】

我要整理的系所/學院描述：
【在這裡貼上學院與系所清單，例如：商管學院：企業管理、財務金融、會計、國際企業、保險、經濟、統計與資料科學。】
```

### 關鍵字擴充 Prompt

```text
你是一位圖書館電子書選書與學科分類助理。請根據我提供的系所清單，為每個系所擴充可用於 Wiley 英文書目比對的關鍵字。

輸出規則：
1. 只輸出 CSV，不要 Markdown，不要說明文字。
2. 第一列固定為：學校名稱,系所名稱,關鍵字
3. 關鍵字用半形分號 ; 分隔。
4. 每個系所至少產生 12 個關鍵字。
5. 關鍵字要包含中文與英文。
6. 英文關鍵字要包含學科正式名稱、常見書名詞、研究主題詞。
7. 避免太廣泛的詞，例如 research、introduction、theory、practice、management 單獨使用時太泛者不要單獨列入，除非搭配具體領域。
8. 不要加入其他學校的系所。
9. 不要改變學校名稱與系所名稱。

請依照以下資料產生：
學校名稱,系所名稱
【在這裡貼系所清單】
```

## 部署到 GitHub Pages

把這些檔案放到 GitHub repository：

```text
index.html
README.md
使用說明.txt
系所關鍵字範本.csv
AI產生系所關鍵字Prompt.txt
```

到 GitHub repository 的 `Settings > Pages` 啟用 GitHub Pages，即可用網址開啟。

`index.html` 會從 CDN 載入 SheetJS Excel 元件，所以 repository 不需要放 `xlsx.full.min.js`。

## 隱私

所有 Excel 和 CSV 都在使用者瀏覽器本機處理，不會上傳到伺服器。
